## Installation

<Tabs>

<Tab title="CLI">

```sh CLI
opengb module add auth_email
```

</Tab>

<Tab title="backend.json">

Add the following to your `backend.json`:

```json backend.json
"modules": {
	"auth_email": {
		"config": {
			// Your config here. See config docs for more details.
		}
	}
}

```
</Tab>

### Internal

These scripts can only be called by other OpenGB modules.

<CardGroup><Card title="Send Email Verification (No Password)" href="/modules/auth_email/scripts/send_verification">Send a one-time verification code to an email address to verify ownership. Does not require a password.</Card><Card title="Sign In with Email and Password" href="/modules/auth_email/scripts/sign_in_email_pass">Sign in a user with an email and password.</Card><Card title="Verify and Add Email and Password to existing user" href="/modules/auth_email/scripts/verify_add_email_pass">Verify a user's email address and register it with an existing account. Requires a password.</Card><Card title="Verify and Add Email to Existing User (No Password)" href="/modules/auth_email/scripts/verify_add_no_pass">Verify a user's email address and register it with an existing account. Does not require a password.</Card><Card title="Verify and Link Email Address to User" href="/modules/auth_email/scripts/verify_link_email">Verify a user's email address and link it to an existing account without allowing login passwordless.</Card><Card title="Verify and Login as (or Create) User (No Password)" href="/modules/auth_email/scripts/verify_login_or_create_no_pass">Verify the email address code and return a userToken to AN account (creates a new account if one doesn't exist). Does not require a password.</Card><Card title="Verify and Sign Up with Email and Password" href="/modules/auth_email/scripts/verify_sign_up_email_pass">Sign up a new user with an email and password.</Card></CardGroup>

- **Verification Code Expired** (`verification_code_expired`) 
- **Verification Code Invalid** (`verification_code_invalid`)
